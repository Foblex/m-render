# Building Example Pages

Example pages are useful when you want to verify features independently and avoid mixed scenarios.

## Recommended structure

1. One feature per page.
2. One short validation checklist per page.
3. Reusable links from an overview page.

## Current examples in docs

* TOC off
* TOC off + wide content
* SEO overrides
* `ng-component` demo
* code-group and alerts

## Reuse pattern

The same format works for blog and releases. Content changes, rendering stack stays the same.
